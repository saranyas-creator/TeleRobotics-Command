# Telemetry Service

## Overview

The **Telemetry Service** provides the shared memory communication layer for the Qt application.

Robot telemetry is generated by the **Software Services Communication Layer**, where the **UDP RX Loop** continuously receives telemetry packets from the robot. After processing each packet, the **SharedTelemetryWriter** publishes the latest robot telemetry into a shared memory region.

The Telemetry Service attaches to this shared memory and provides the latest robot telemetry to the Qt application.

Unlike the communication workers, the Telemetry Service does not use ZeroMQ and is not managed by the `ZmqManager`. Instead, it provides direct read access to shared memory, enabling low-latency communication between the Software Services and the Qt application.

### Responsibilities

- Connect to the shared memory region.
- Read the latest robot telemetry.
- Verify shared memory consistency.
- Verify telemetry freshness.
- Return telemetry snapshots to the Qt application.

---

# Architecture

The Telemetry Service acts as the bridge between the Software Services and the Qt application.

```text
                           Qt Application
                                  │
                                  ▼
                          Telemetry Service
                                  │
                                  ▼
                           Shared Memory
                                  ▲
                                  │
                      SharedTelemetryWriter
                                  ▲
                                  │
                            UDP RX Loop
                                  ▲
                                  │
                          Raw UDP Channel
                                  ▲
                                  │
                                 Robot
```

The telemetry communication sequence is:

1. Robot transmits telemetry packets.
2. Raw UDP Channel receives the packets.
3. UDP RX Loop extracts the telemetry.
4. SharedTelemetryWriter publishes the telemetry into shared memory.
5. Telemetry Service reads the shared memory.
6. Qt application retrieves the latest telemetry.

# 1. Telemetry Service API

The Telemetry Service exposes a simple interface for accessing robot telemetry.

```cpp
TelemetryService& instance();

bool attach();

TelemetrySnapshot getLatest();
```

Each function has a dedicated responsibility for connecting to shared memory and retrieving robot telemetry. :contentReference[oaicite:1]{index=1}

---

## 1.1 Singleton Instance

The Telemetry Service is implemented as a singleton.

```cpp
TelemetryService::instance();
```

Only one Telemetry Service exists within the Qt application.

This allows all Qt modules to access the same shared memory region without creating multiple shared memory mappings.

```text
VTK Mesh Viewer
        │
        ▼
TelemetryService::instance()
        ▲
        │
Other Qt Modules
```

---

## 1.2 Attaching to Shared Memory

Before telemetry can be read, the service must attach to the shared memory region.

```cpp
bool attach();
```

Internally, the service:

- Opens the shared memory object.
- Maps the shared memory into the Qt process.
- Stores the mapped memory pointer.

```text
attach()
     │
     ▼
shm_open()
     │
     ▼
mmap()
     │
     ▼
Ready to Read
```

If the shared memory is unavailable, the service reports the failure and retries when telemetry is requested. :contentReference[oaicite:2]{index=2}

## 1.3 Reading Telemetry

The latest robot telemetry is obtained using:

```cpp
TelemetrySnapshot getLatest();
```

This function reads the current telemetry stored in shared memory and returns it as a `TelemetrySnapshot`.

Workflow:

```text
getLatest()
      │
      ▼
Read Shared Memory
      │
      ▼
Verify Consistency
      │
      ▼
Verify Freshness
      │
      ▼
Return TelemetrySnapshot
```

The returned snapshot contains:

- STL Position
- STL Orientation
- Force Feedback
- Telemetry Age
- Valid Flag

If the shared memory is not attached, the service automatically attempts to attach before reading the telemetry.

# 2. Telemetry Snapshot

The Telemetry Service returns robot telemetry using the `TelemetrySnapshot` structure.

```cpp
struct TelemetrySnapshot
{
    double pos[3];
    double ang[3];
    double force[3];

    qint64 ageNs;
    bool valid;
};
```

The snapshot represents a consistent copy of the latest robot telemetry.

| Field | Description |
|--------|-------------|
| `pos[3]` | STL Position (X, Y, Z) |
| `ang[3]` | STL Orientation (Roll, Pitch, Yaw) |
| `force[3]` | Force Feedback (X, Y, Z) |
| `ageNs` | Age of the telemetry |
| `valid` | Indicates whether the telemetry is valid |

# 3. Shared Memory Consistency

The shared memory is continuously updated by the Software Services while it is being read by the Qt application.

To ensure that telemetry is not modified during a read operation, the Telemetry Service uses the sequence number stored in shared memory.

```text
Read Sequence Number
        │
        ▼
Copy Telemetry
        │
        ▼
Read Sequence Number Again
        │
        ▼
Sequence Changed?
      │        │
     No       Yes
      │        │
      ▼        ▼
 Return    Retry Read
```

This mechanism guarantees that the Qt application always receives a consistent telemetry snapshot rather than partially updated data.

---

# 4. Telemetry Freshness

After reading the telemetry, the service checks whether the received data is recent.

The timestamp stored in shared memory is compared with the current system time.

If the telemetry exceeds the configured freshness threshold, the snapshot is marked as invalid.

If telemetry remains stale for an extended period, the service automatically detaches from the shared memory. The next telemetry request attempts to establish a new shared memory mapping. :contentReference[oaicite:6]{index=6}

# 5. Qt Integration

The Qt application retrieves robot telemetry through the Telemetry Service.

```text
                  Qt Application
                        │
                        ▼  
                Telemetry Service
                        │
                        ▼
              Telemetry Snapshot
                        │
        ┌───────────────┼────────────────┐
        ▼                                ▼
  VTK Mesh Viewer                 Force Monitoring
        │
        ▼
  STL Probe Position
  STL Probe Orientation
```

The telemetry snapshot is used by the Qt application to update the probe visualization.

- **STL Position (X, Y, Z)** – Updates the position of the 3D probe model.
- **STL Orientation (Roll, Pitch, Yaw)** – Updates the orientation of the 3D probe model.
- **Force Feedback (X, Y, Z)** – Provides the latest robot force measurements for visualization or future application modules.

By centralizing shared memory access within the Telemetry Service, all Qt components retrieve telemetry through a single, consistent interface.

# Summary

The Telemetry Service provides the shared memory communication layer for the Qt application.

It reads the latest robot telemetry from the shared memory published by the Software Services, verifies the consistency and freshness of the data, and returns validated telemetry snapshots to the Qt application.

The retrieved STL position and orientation are used to update the 3D probe model displayed in the VTK mesh viewer, enabling real-time visualization of the robot pose.
